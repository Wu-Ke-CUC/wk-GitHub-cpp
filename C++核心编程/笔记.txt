内存四区：代码区，全局区，栈区，堆区
程序运行前{
	代码区：存放二进制机器指令		共享，只读
	全局区：存放全局变量，静态变量，常量	程序结束时自动释放
}
程序运行后{
	栈区：存放函数形参，局部变量	不要返回局部变量地址
	堆区：程序员分配和释放		程序结束时自动释放	使用new开辟数据（开辟的是内存，例：int* p=new int(10)，使用delete释放)
		  指针本质是局部变量，存放在栈区，指针保存的数据是放在堆区
}
引用{
	给变量取别名	本质是指针常量(数据类型* const 别名 = &原名;)
	语法：数据类型 &别名=原名;	//引用必须要初始化，求初始化后不能改变
	函数传参时，可以用引用使形参修饰实参
	引用做函数返回值，不能返回局部变量的引用，函数的调用可以作为函数的左值
	常量引用：(const int& ref = 10;)==(int temp=10; int& ref = temp;)//ref只读
			  作为形参可防止改变实参
}
函数高级{
	函数默认参数：默认参数在形参列表结尾；函数的声明和实现不能同时有默认参数
	占位参数	：参数只有数据类型，可以有默认参数
	函数重载	：同一作用域下，函数名相同，参数数量，类型或顺序不同；函数重载中有默认参数容易出现二义性
}

面向对象三大特性：封装，继承，多态
封装{
	意义：将属性和行为作为一个整体，并加以权限控制
	访问权限(默认为私有)：public   (类内外都可访问)	
						  protected(类内可以访问，类外不可访问)(子类的类内可访问父类)
						  private  (类内可以访问，类外不可访问)
	成员属性（成员变量）	成员函数（成员方法）
}
对象的初始化和清理{
	构造函数和析构函数：编译器自动调用		构造函数：类名(){}--(可以有参数)		析构函数：~类名(){}--(不能有参数)
	构造函数：按参数分类（有参构造，无参构造（默认））	按类型分（普通构造，拷贝构造（默认浅拷贝））	调用方式：括号法，显示法，隐式转换法
			  类名(参数)--（匿名对象）	拷贝构造包括浅拷贝（容易造成内存重复释放）和深拷贝
			  其它类的对象作为类对象的属性时，构造时其他类会先构造，析构时会后析构
	静态成员：编译阶段分配内存，类内声明，类外初始化
			  所有对象共用同一份数据与同一个函数，可通过对象和类名访问变量或函数（有权限管理）
			  静态成员函数只能访问静态成员变量
}
对象模型和this指针{
	空对象占用内存空间为1	成员变量和成员函数分开储存
	静态变量，函数和静态函数不属于类对象上
	this指针指向被调用的成员函数所属的成员变量->用于解决名称冲突和返回对象本身(类型为引用类型)
	无法访问空指针的属性	this指针时指针常量，无法修改	成员函数后面加const(常函数)，则属性也无法修改，除非属性有mutable关键字
	常对象：对象前面加const修饰，只能调用常函数
}
友元{
	目的是让一个函数或者类访问另一个类的私有成员	关键字为friend
	全局函数，类和成员函数都可以做友元
}
运算符重载{
	运算符重载也可以进行重载
	类()--匿名对象，使用完销毁
}
继承{
	class 子类 : 继承方式 父类（可继承多个类）
	继承方式：公共继承（权限不变），保护继承（共有变保护），私有继承（共有和保护变私有）
	父类所有非静态属性都会被子类继承，但是私有的被编译器隐藏了，所以访问不到
	创建子类对象时也会调用父类的构造函数(构造时先构造父类，析构时先析构子类)
	成员属性若与父类重名，则调用父类属性需要添加作用域，静态成员变量和非静态成员变量一致
	虚继承解决菱形继承问题，确保同名数据只有一个
}
